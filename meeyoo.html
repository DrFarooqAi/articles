Core Features

Built around radical privacy

Every decision in Mee&Yoo is made with one principle: the server should never know what you're saying.

🔐
AES-256-GCM Encryption

Military-grade encryption happens entirely in your browser using the Web Crypto API. Your message is locked before it ever leaves your device.

PBKDF2 · 100k iterations · SHA-256
🧠
RAM-Only Messages

No database. No disk writes. Messages live in server memory only — cleared after delivery, wiped on Ctrl+C, erased when you close the tab.

No SQL · No Logs · No Persistence
👤
Zero Accounts

No sign-up. No email. No phone number. Just share the link and a shared passphrase once — then chat privately, instantly.

No Registration · No Identity
🕶️
Blind Relay Server

The server only routes ciphertext. It never sees your passphrase, never derives a key, never reads plaintext. Zero-trust by architecture.

Ciphertext-Only · Zero-Trust
💨
Ephemeral by Design

Close the tab — keys are erased. Stop the server — memory is cleared. No message can be recovered after a session ends. Ever.

Tab Closed → Gone Forever

Zero Dependencies

01
Share the link & a secret word

Send your partner the server link and agree on a shared passphrase (e.g. "sunflower"). Your browsers each derive the same AES key from it using PBKDF2 — the key never leaves your devices.

passphrase → PBKDF2(100k, SHA-256) → AES-256-GCM key
02
Type — your browser encrypts before sending

Every message is locked with a random 12-byte IV and your AES key inside your browser before it touches the network. The server receives only Base64 ciphertext. It has no idea what you said.

lock(plaintext) → AES-GCM(IV + ciphertext) → POST /api/send
03
Close the tab — everything vanishes

When your partner's browser polls and receives the message, it's decrypted locally. The server immediately wipes its queue. Close the tab and your AES key is erased from memory. No one can recover it.

Tab closed → key erased → server wipes → RAM cleared → gone

The Privacy Promise

What we never do

Four absolute guarantees — not policies, not terms of service, but architectural impossibilities.

🚫
No Database

No SQL, no NoSQL, no file system. Messages exist in a Python list in RAM and nowhere else.

🚫
No Logs

No server-side logging of messages, IPs, or sessions. No analytics. No telemetry. Nothing is written to disk.

🚫
No Accounts

No username, email, phone, or identity of any kind required or stored. You are anonymous by design.

🚫
No Plaintext

The server receives only ciphertext. It has no passphrase, no key, no ability to decrypt what you send. Ever.
